WikiOasis `mw-config` is the production configuration layer that powers the entire WikiOasis MediaWiki farm. It manages hundreds of wikis across `wikioasis.org` and `skywiki.org`, providing per-wiki settings, global extension loading, database cluster routing, object storage, caching, and error handling — all forked and extended from Miraheze's battle-tested `mw-config` architecture.

## How the farm works

The WikiOasis farm bootstraps each request through a layered configuration pipeline:

<Steps>
  <Step title="Request arrives">
    MediaWiki loads `LocalSettings.php`, which detects the current wiki from the request hostname and database name.
  </Step>
  <Step title="MirahezeFunctions initializes">
    `$wi = new MirahezeFunctions()` resolves the DB name, realm (production vs. beta), MediaWiki version, server URL, and wiki state from the CreateWiki cache.
  </Step>
  <Step title="SiteConfiguration applies per-wiki settings">
    `$wgConf->settings` maps thousands of per-wiki overrides for permissions, uploads, skins, extensions, and more — resolved by `$wi->dbname`.
  </Step>
  <Step title="GlobalSettings and extensions load">
    `GlobalSettings.php` applies farm-wide settings (CentralAuth, OpenSearch, Discord notifications, DataDump), and `GlobalExtensions.php` loads all globally required extensions and skins.
  </Step>
</Steps>

## Key capabilities

<CardGroup cols={2}>
  <Card title="Two Realms" icon="layer-group">
    Production (`wikioasis.org` / `skywiki.org`) and beta (`betaoasis.xyz`) realms with independent database clusters and configuration.
  </Card>
  <Card title="CentralAuth" icon="user-shield">
    Strict global accounts with SUL3 support and a shared domain (`accounts.wikioasis.org`) for cross-wiki authentication.
  </Card>
  <Card title="R2 Object Storage" icon="cloud">
    All wiki media stored in Cloudflare R2 (`wikioasis-media` bucket) with CDN delivery via `cdn.wikioasis.org`.
  </Card>
  <Card title="Per-Wiki Versioning" icon="code-branch">
    Individual wikis can be pinned to specific MediaWiki versions via `WikiFarmMultiVersion` without affecting the rest of the farm.
  </Card>
  <Card title="ManageWiki Integration" icon="sliders">
    Wiki admins self-serve extension activation, namespace configuration, and settings through ManageWiki — no config file edits needed.
  </Card>
  <Card title="Observability" icon="chart-line">
    Sentry error tracking with 100% trace sampling, DogStatsD metrics to monitoring hosts, and Monolog stderr logging.
  </Card>
</CardGroup>
